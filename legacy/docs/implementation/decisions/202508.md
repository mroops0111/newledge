2025-08-10:
* [Auth] Prefer HttpOnly cookie for access token
  * [Decision] 前端不再使用 localStorage 儲存 access token，統一透過 Next.js API route 設定的 HttpOnly cookie 帶到後端
  * [Reasoning] 降低 XSS 風險，與後端授權模型一致
* [Integration] 後端代理層
  * [Decision] 以 `/api/tracks`, `/api/track-domains`, `/api/track-sources` 作為後端 API 的 proxy，集中處理驗證、錯誤與快取策略
  * [Reasoning] 隔離環境變數、簡化前端呼叫、方便日後加入觀測/重試
* [Model Mapping]
  * [Decision] 在 `TrackService` 進行後端模型到 UI 模型的對應（domain/name 與 sources 陣列的轉換）
  * [Reasoning] 讓 UI 維持簡潔、避免在各元件散落轉換邏輯
* [Cleanup] 刪除未使用主題功能相關碼
  * [Decision] 移除 `TopicForm`, `TopicList`, `types/Topic.ts`，暫緩主題功能設計
  * [Reasoning] 當前 Roadmap 聚焦 Track 管理與登入；減少死碼與維護成本
* [Cleanup] 刪除未使用的 `authService`
  * [Decision] 採用純 Cookie 模式與 Next.js API route；不再於前端以 `authService` 處理 OAuth
  * [Reasoning] 降低前端複雜度與安全風險
